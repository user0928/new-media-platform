## 2. 本阶段目标

本阶段围绕“让新媒体协作平台真正可用”进行前端 Demo 优化，重点不是增加静态展示，而是补齐日常使用中最关键的任务流转：

- 各部门能提交自己的交付物。
- 技术部能提交封面图片。
- 成员能看到自己的任务。
- 主任、老师能发布任务。
- 部门负责人能审核本部门提交。
- 老师、主任和开发人员能查看并处理全部部门事项。

## 3. 主要设计决策

### 3.1 侧边栏精简

早期页面入口较多，容易让用户不知道从哪里开始处理任务。因此侧边栏保留为高频入口：

- 工作台
- 我的
- 项目看板
- 部门任务
- 成员权限

项目详情不再作为常驻侧边栏入口，而是从任务、看板、审批等上下文进入。

### 3.2 默认开发者最高权限

当前处于 Demo 和开发阶段，为方便测试，默认登录身份设置为：

- 用户：`Codex 开发者`
- 角色：`开发人员`
- 权限：默认最高权限

开发人员可以查看全部部门任务、全部提交、全部审核队列，并可模拟老师、主任、负责人、干事的关键操作。

### 3.3 提交与审核拆成明确链路

提交不再只是生成一条记录，而是进入审核状态：

```text
部门成员提交
  -> 进入待审核
  -> 部门负责人 / 主任 / 老师 / 开发人员审核
  -> 通过或驳回
```

这样技术部上传封面图片后，负责人可以立即在审核队列中看到并处理。

| 角色 | 可查看范围 | 可审核范围 | 可发布任务 |
| --- | --- | --- | --- |
| 开发人员 | 全部部门 | 全部提交 | 可以 |
| 负责老师 | 全部部门 | 全部提交 | 可以 |

## 6. 当前核心流程

### 6.1 技术部提交封面图片

```text
进入“部门任务”
  -> 选择技术部任务
  -> 提交类型默认为“图片文件”
  -> 点击选择图片或拖拽上传
  -> 提交给负责人审核
  -> 提交动态显示“待审核”
```

### 6.2 部门负责人审核

```text
以部门负责人身份登录
  -> 进入“我的”
  -> 查看“待我审核”
  -> 通过或驳回本部门提交
```

### 6.3 高级角色总览

```text
以开发人员 / 老师 / 主任身份登录
  -> 进入“部门任务”
  -> 可选择“全部部门”
  -> 查看全部任务和提交动态
  -> 可审核任意部门提交
```
